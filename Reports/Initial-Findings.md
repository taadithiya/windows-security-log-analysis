\# Initial Windows Security Log Findings



\## Dataset Summary



\- Collection period: Previous seven days

\- Total events analysed: 8,412

\- Data source: Windows Security Event Log

\- Collection method: PowerShell

\- Format: CSV



\## Event Distribution



| Event ID | Count | Description |

|---|---:|---|

| 5379 | 7,053 | Credential Manager credentials were read |

| 4624 | 444 | Successful account logon |

| 4672 | 427 | Special privileges assigned to a new logon |

| 4799 | 135 | Security-enabled local group membership enumerated |

| 4798 | 77 | User local group membership enumerated |

| 5382 | 63 | Vault credentials were read |

| 4634 | 25 | Account logged off |

| 4648 | 20 | Logon attempted using explicit credentials |



\## Authentication Findings



A total of 444 successful-logon events were identified using Event ID 4624.



No Event ID 4625 failed-logon events were identified in the collected dataset. This means no failed-logon records were captured during the reviewed period. Audit configuration and log retention should be considered when interpreting this result.



\## Privileged Logon Findings



A total of 427 Event ID 4672 records were identified. These events indicate that sensitive privileges were assigned to new logon sessions. Such events are often associated with administrators, system accounts and Windows services.



\### Privileged Logon Account Analysis



The 427 Event ID 4672 records were grouped by account name.



| Account Category | Count | Interpretation |

|---|---:|---|

| SYSTEM | 407 | Privileged activity associated with the Windows operating system |

| Personal user account | 14 | Privileged logons associated with the local user |

| Desktop Window Manager accounts | 6 | Activity associated with Windows graphical session accounts |



Most privileged logons were generated by the SYSTEM account. A smaller number were associated with the personal user account and Desktop Window Manager session accounts.



The personal account name was redacted from public screenshots for privacy.



\## Credential Manager Activity



Event ID 5379 was the most frequently recorded event, with 7,053 occurrences. This event indicates that credentials stored in Windows Credential Manager were read.



The high volume should be investigated by reviewing account names, credential targets, processes and timing. Frequency alone does not establish malicious activity.



\## Credential Manager Account Analysis



The 7,053 Event ID 5379 records were grouped by account name.



| Account Category | Count | Interpretation |

|---|---:|---|

| Local user account | 6,807 | Primary user-context activity |

| Computer account | 226 | Activity associated with the Windows computer account |

| LOCAL SERVICE | 20 | Activity generated by a built-in Windows service account |



The local user account generated most Credential Manager access events. Smaller volumes were associated with the computer account and the built-in LOCAL SERVICE account.



This distribution is consistent with routine Windows and application credential access, although account frequency alone is not sufficient to classify the activity as benign. Further correlation with credential targets, processes and timestamps is recommended.



The actual account and computer names were redacted from public screenshots for privacy.



\## Successful Logon Type Analysis



A total of 444 successful logon events were identified and classified by logon type.



| Logon Type | Count | Description |

|---|---:|---|

| 5 | 407 | Service logon |

| 11 | 14 | Cached interactive logon |

| 7 | 14 | Workstation unlock |

| 2 | 9 | Interactive local logon |



The majority of successful logons were Type 5 service logons, accounting for 407 of the 444 successful authentication events. This indicates that most authentication activity was generated by Windows services or background processes.



Fourteen Type 11 cached interactive logons were identified. Fourteen Type 7 workstation-unlock events and nine Type 2 interactive local logons were also recorded.



No Type 10 Remote Desktop logons were identified in the reviewed dataset.



\## Audit Log Clearing



No Event ID 1102 records were found. Therefore, no Security audit-log clearing activity was observed in the reviewed dataset.



\## Limitations



\- The analysis covers only the exported seven-day period.

\- Some security events may not have been generated if the required audit policies were disabled.

\- Event counts alone cannot determine whether an activity is malicious.

\- Events should be correlated using account, process, source address, logon type and timestamp.

\- Credential-target details were not available in the rendered Event ID 5379 CSV messages.



\## Initial Conclusion



The dataset primarily contains Credential Manager access, successful authentication and privileged-logon activity.



Successful logons were predominantly associated with Windows services. No Remote Desktop logons, failed logons or Security-log-clearing events were observed.



Further investigation should focus on the accounts and processes associated with service logons and Credential Manager access.
